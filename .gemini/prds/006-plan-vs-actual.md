# PRD: 계획 대비 성취도 분석 (Plan vs. Actual)

## Metadata
- **Status**: Draft
- **Depends On**: PRD-002, PRD-005

## 1. 개요
계획(Plan)과 실제 수행(Actual)을 비교하여 얼마나 잘 지켰는지 평가하는 시스템입니다.

## 2. 매칭 및 평가 로직

### 2.1 매칭 (Matching)
-   **Key**: `Date` (날짜가 같으면 매칭 시도).
-   **다대다 처리**: 하루에 계획 2개, 활동 2개일 경우 -> 가까운 시간대 또는 거리로 매칭 (V1은 1:1 매칭만 지원).

### 2.2 성취도 등급 (Compliance Grade)
-   **Perfect (초록)**: 거리 오차 ±5% 이내 AND 페이스 오차 ±5% 이내.
-   **Good (노랑)**: 거리 오차 ±15% 이내.
-   **Missed (빨강)**: 계획만 있고 활동 없음.
-   **Unplanned (회색)**: 계획 없이 뛴 경우.

## 3. UI 시각화
-   캘린더 셀의 배경색을 성취도 등급 색상으로 변경.
-   클릭 시 팝오버로 "목표: 10km vs 실제: 9.8km" 상세 비교 표시.

## 4. 성공 기준
-   [ ] 데이터가 로딩되면 클라이언트 사이드에서 즉시 매칭이 이루어져야 함.
-   [ ] 시각적으로 성취도가 한눈에 구분되어야 함.
