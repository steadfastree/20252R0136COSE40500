# PRD: 통합 대시보드 및 활동 로그 (Dashboard & UI)

## 1. 개요
PRD-002에서 수집하고 PRD-003에서 분석한 데이터를 사용자에게 직관적으로 전달합니다. 러너들이 가장 궁금해하는 "나의 현재 상태"와 "훈련 기록"을 한눈에 파악하고, 깊이 있게 탐색할 수 있는 인터페이스를 제공합니다.

## 2. 주요 구성 요소

### 2.1 KPI 카드 (Key Performance Indicators)
상단에 배치하여 핵심 지표를 즉각적으로 전달합니다.
- **VDOT 레벨**: 현재 내 러닝 레벨과 예상 기록 (5K, 10K, Half, Full).
- **이번 주 부하 (Load)**: 이번 주 달린 거리(km) 및 지난주 대비 증감율.
- **ACWR 상태**: 부상 위험도 신호등 (초록/노랑/빨강).
- **꾸준함 점수**: 최근 훈련의 성실도 점수.

### 2.2 메인 차트 (Main Visualization)
- **주간 훈련량 추이**: 최근 12주간의 거리(Bar)와 ACWR(Line)을 겹쳐서 시각화 (Recharts 활용).
- **효율성 변화**: 효율성 지수(Efficiency Index)의 변화 추세를 꺾은선 그래프로 표시.

### 2.3 활동 로그 그리드 (Activity Grid)
AG Grid를 활용하여 엑셀처럼 강력한 데이터 탐색 기능을 제공합니다.
- **필터링**: 거리순, 페이스순, 날짜순 정렬 및 필터.
- **컬럼 설정**: 사용자가 보고 싶은 데이터(심박수, 케이던스, 고도 등)를 선택.
- **상세 보기**: 행을 클릭하면 해당 활동의 Strava 페이지로 이동하거나 상세 모달 표시.

## 3. 기술적 요구사항

### 3.1 상태 관리 (Zustand)
- 대시보드의 필터 상태(기간 설정, 활동 타입 등)는 Zustand 전역 스토어에서 관리하여, 페이지 이동 후 돌아와도 상태가 유지되도록 함.

### 3.2 반응형 디자인 (Responsive)
- 모바일에서는 KPI 카드와 요약 차트 위주로 표시.
- 데스크탑에서는 AG Grid를 포함한 전체 데이터 탐색 뷰 제공.

## 4. 디자인 시스템 (Design System)
- **Shadcn/UI**와 **Radix UI**를 적극 활용하여 일관된 룩앤필 유지.
- **색상 테마**: Strava의 오렌지색(#FC4C02)을 포인트 컬러로 사용하되, 전반적으로는 눈이 편안한 Slate/Zinc 계열 사용.

## 5. 성공 기준
- 대시보드 로딩 속도 최적화 (TanStack Query 캐싱 활용).
- AG Grid가 1,000개 이상의 활동 데이터도 버벅임 없이 렌더링해야 함.
- 모바일/데스크탑 뷰가 깨짐 없이 동작해야 함.
